# 보안/개인정보 관점에서 사용자 우려 가능 포인트 정리

이 문서는 **현재 코드 기준**으로 사용자가 앱을 사용할 때 우려할 수 있는 보안·개인정보 이슈를 정리한 메모입니다.

## 1) 로컬에 저장되는 데이터(다운로드 이력/큐/설정)

앱은 로컬 SQLite DB(`ytdlp.db`)와 설정 파일(`settings.json`)에 다음 정보를 저장합니다.

- 다운로드 큐/이력: 영상 URL, 영상 ID, 제목, 저장 경로, 오류 메시지, 다운로드 시각
- 설정: 다운로드 경로, 파일명 템플릿, 쿠키 브라우저 선택, 언어/테마 등

사용자 입장 우려:

- PC를 함께 쓰는 환경에서 **시청/다운로드 관심사**가 노출될 수 있음
- 파일 경로/제목 등 메타데이터가 장기간 남아 프라이버시 부담이 생길 수 있음

## 2) 쿠키 브라우저 기능 사용 시 계정 정보 노출 가능성

앱은 설정된 브라우저 쿠키를 `yt-dlp --cookies-from-browser`로 전달해 로그인 필요한 콘텐츠를 처리합니다.

사용자 입장 우려:

- 로그인 세션 기반 동작이라, 민감 계정으로 사용할 경우 심리적/실질적 위험 인식이 큼
- 잘못된 사용(공용 PC, 업무 계정 등) 시 개인정보·계정 접근 우려

## 3) 로그 파일(`log.txt`)에 민감 문자열이 남을 가능성

앱 로거는 앱 데이터 디렉터리에 `log.txt`를 남기며, 에러 문구를 기록합니다.

사용자 입장 우려:

- 오류 상황에 따라 URL/경로/시스템 정보 일부가 로그에 포함될 수 있음
- 로그 정리 기능이 없으면 데이터 잔존 기간이 길어짐

## 4) 자동 업데이트/외부 바이너리 실행에 대한 신뢰 이슈

앱은 시스템의 `yt-dlp`를 실행하고, 업데이트 명령(`--update`)도 호출합니다.

사용자 입장 우려:

- "외부 바이너리를 실행한다"는 점 자체가 공급망/무결성 관점에서 부담
- 업데이트 결과·출처 검증을 사용자가 체감하기 어려움

## 5) Tauri 보안 설정(CSP 비활성화)

현재 설정에서 `app.security.csp`가 `null`입니다.

사용자 입장 우려:

- 웹뷰 보안 정책이 느슨하다고 받아들일 수 있음
- 실제 악용 여부와 별개로, 보안 민감 사용자에게 불안 포인트가 됨

## 6) URL 입력 기반 동작으로 인한 사회공학/오남용 우려

앱은 사용자가 입력한 URL을 대상으로 메타데이터 조회/다운로드를 수행합니다.

사용자 입장 우려:

- 신뢰하지 않는 URL 입력 시 예기치 않은 콘텐츠/트래픽 발생 가능성
- 저작권/서비스 약관 위반 리스크를 개인정보 리스크와 함께 걱정할 수 있음

---

## 사용자 안내 문구에 추가하면 좋은 내용(권장)

- "이 앱은 다운로드 이력/설정을 로컬에 저장합니다"를 명시
- "쿠키 브라우저는 로그인 세션을 활용하므로 개인 기기에서만 사용" 경고
- "공용 PC에서는 이력 삭제/로그 삭제 후 종료" 가이드
- "자동 업데이트를 끄거나, 신뢰 가능한 설치 방식만 사용" 선택지 제공
- 개인정보 최소화를 위해
  - 이력 자동 삭제(기간 기반)
  - 로그 비활성화/마스킹
  - 민감 필드 저장 최소화(예: 전체 URL 저장 옵션화)
  를 추후 옵션으로 제공
